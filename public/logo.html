[USAGE_RULES]

Rules Claude must follow when generating UI from this spec

  1. ALWAYS use --sp- prefixed CSS variables, never hardcode hex values
  2. NEVER use rgba(0,0,0,...) for shadows — use the sp-shadow tokens only
  3. NEVER use border-radius values not in the radius scale
  4. Primary CTA buttons MUST use gradient_primary + shadow_2 minimum
  5. Left-accent borders on notifications MUST be 3px with border-radius: 0
  6. The 2px border_accent is ONLY for featured/highlighted card outlines
  7. Progress bars with active fills MUST include matching glow shadow
  8. Elevation increases on hover — never decrease elevation on hover
  9. Gradient surfaces must carry a matching blue-tinted shadow
  10. Acrylic/glass cards are for contextual info only — never for forms or data tables
  11. Avatar stacks overlap at margin-left: -8px with white border: 2px
  12. All interactive elements have transition: all var(--sp-dur-normal) var(--sp-ease)
  13. Disabled states: opacity 0.45, cursor: not-allowed — never change color
  14. Font is Nunito; fallback to Segoe UI — NEVER use Inter, Roboto, or Arial as primary
  15. Status badges always include a colored dot before the label text

[RESPONSIVE_DESIGN]

Added: 2026-04-26 | Version bump: 1.1.0

Rule: ALL S-planned UI must be fully responsive across the three defined breakpoints.

Claude must apply these rules to every component and layout it generates.

Breakpoints

mobile: max-width: 639px # phones, small screens tablet: 640px – 1023px # tablets, large phones landscape desktop: 1024px and above # laptops, monitors (primary design target)

CSS breakpoint variables (use in media queries)

--bp-mobile: 639px --bp-tablet: 1023px --bp-desktop: 1024px

media_query_mobile: @media (max-width: 639px) media_query_tablet: @media (min-width: 640px) and (max-width: 1023px) media_query_desktop: @media (min-width: 1024px)

Grid system — responsive

desktop: columns: 12 gutter: 24px max_width: 1280px margin: auto (centered)

tablet: columns: 8 gutter: 16px max_width: 100% margin: 0 24px

mobile: columns: 4 gutter: 12px max_width: 100% margin: 0 16px

App shell — responsive layout

desktop: layout: sidebar (220px fixed) + main content + optional right panel (220px) sidebar: always visible, full height right_panel: visible when context is available

tablet: layout: collapsed sidebar (icon-only, 56px) + main content sidebar: icons only — no labels, no section titles right_panel: hidden — surfaced as a bottom sheet or slide-over drawer top_bar: breadcrumb collapses to single current page label

mobile: layout: NO sidebar — bottom navigation bar (5 tabs max) sidebar: replaced by bottom nav bar (56px fixed at bottom) right_panel: hidden — surfaced as a full-screen sheet top_bar: back button + page title + overflow menu (3-dot) content: full width, 16px horizontal padding

Bottom navigation bar (mobile only)

height: 56px background: #FFFFFF border_top: 0.5px solid gray_200 shadow: 0 -4px 12px rgba(23,87,194,0.06) items: max 5 tabs item_layout: icon (24px) centered + label below (10px Nunito 700) active_item: { icon_color: blue_500, label_color: blue_500, dot indicator above icon } inactive_item: { icon_color: gray_400, label_color: gray_400 }

tabs:

  • Dashboard
  • Plans
  • Checklists
  • Evidence
  • More (overflow)

Sidebar — tablet (icon-only collapsed)

width: 56px background: #0F1E3D padding: 12px 0 icons: centered, 24x24px, gray_400 inactive / blue_400 active tooltip: show label on hover (desktop-style tooltip) brand: show icon only (no wordmark) user: show avatar only (no name/role)

Spacing — responsive adjustments

desktop: content_padding: 24px card_padding: 20px 24px section_gap: 20px component_gap: 12px

tablet: content_padding: 20px card_padding: 16px 20px section_gap: 16px component_gap: 10px

mobile: content_padding: 16px card_padding: 14px 16px section_gap: 14px component_gap: 8px

body: 14px (unchanged across all breakpoints) caption: 12px (unchanged) overline: 11px (unchanged)

Component responsive rules

Stat / metric cards

desktop: 4-column grid, gap: 12px tablet: 2-column grid, gap: 10px mobile: 2-column grid, gap: 8px (numbers smaller: 20px / 800)

desktop: full layout — icon + title + badge + progress + footer (avatars + actions) tablet: same as desktop mobile:

  • Title: 15px / 800
  • Badge: moved below title (stacked layout)

  • Avatar stack: max 3 visible

Checklist item

desktop: single row — checkbox + label + assignee + badge + due date tablet: same as desktop mobile:

  • Row 2: horizontal scroll filter pills (no sort/view toggle visible)
  • Sort/view: moved to top bar overflow menu

Progress bar

desktop: label left + track + percentage right (single row) tablet: same mobile: label above track, percentage inline right of label (track full width below)

Navigation (right panel)

desktop: fixed right panel (220px) tablet: hidden — toggle via info icon in top bar → slide-over from right mobile: hidden — accessible via bottom sheet (swipe up or "Details" button)

Top bar / breadcrumb

Buttons

desktop: label + icon, full padding (10px 22px) tablet: same mobile:

  • Primary action: full-width button (width: 100%)
  • Secondary actions: icon-only button (36x36px) with tooltip

Touch targets (mobile)

minimum_touch_target: 44x44px

Even if visually smaller (e.g. a 20px checkbox), the tap area must be 44x44px

Achieved via padding or ::before pseudo-element

interactive_spacing: minimum_gap_between_targets: 8px

Prevents accidental taps on adjacent elements

[RESPONSIVE_USAGE_RULES]

Rules Claude must follow for all responsive UI generation

  1. ALWAYS write mobile-first CSS — base styles for mobile, override up with min-width queries
  2. NEVER use px-based media queries that differ from the three defined breakpoints (639, 1023, 1024px)
  3. ALWAYS use minmax(0, 1fr) in CSS grid — never plain 1fr (prevents overflow)
  4. Sidebar is HIDDEN on mobile — ALWAYS replace with bottom navigation bar
  5. Right panel is HIDDEN on tablet and mobile — surface as bottom sheet or slide-over
  6. ALL touch targets must be minimum 44x44px on mobile — use padding to expand if needed
  7. NEVER use hover-only interactions on mobile — all hover states must have a tap/click equivalent
  8. Floating Action Button (FAB) replaces multi-button toolbars on mobile
  9. Modals become bottom sheets on mobile — border-radius: 20px 20px 0 0
  10. Tables become card-per-row on mobile — never render a horizontal-scroll table
  11. Breadcrumbs collapse to back-arrow + page title on mobile
  12. Font sizes NEVER go below 12px on any breakpoint
  13. Line-height must be minimum 1.4 on mobile for readability
  14. Images and media always have max-width: 100%, height: auto
  15. NEVER hardcode px widths on flex/grid children — use fr units, %, or minmax()
